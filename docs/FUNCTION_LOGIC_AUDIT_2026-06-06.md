## 本轮修复

| 问题 | 判断 | 处理 |
|---|---|---|
| 会话触摸区域和显示区域不一致 | 已修复 | 触摸层覆盖整个预览区域，但 `TouchType.Down` 必须落在已渲染图像内才发输入；移动/抬起只在已有指针时钳制到边界。 |
| 远端坐标可能落到宽高边界外 | 已修复 | `mapTouchToRemote()` 将本地远端坐标限制到 `interactiveWidth - 1` / `interactiveHeight - 1`。 |
| 聊天消息发送后不滚到底 | 已修复 | TAB 聊天、远控浮动聊天、独立聊天页均改为 0ms/延迟/渲染后多次滚动兜底。 |
| 聊天记录状态不刷新 | 已修复 | `ChatService.getMessages()` 返回数组快照，避免内部数组 push 后页面拿到同一引用而不触发刷新。 |
| 已读状态重启后可能恢复未读 | 已修复 | `markAsRead()` 在实际变更后持久化。 |
| 独立聊天页仍有固定文件引用 | 已修复 | `Chat.ets` 的文件引用按钮改为真实文档选择器，不再发送固定 `/remote/Inbox/logs.zip`。 |
| 服务器配置导入只写草稿 | 已修复 | 设置页导入后直接写入 `AppDataService` 并同步 `NativeRustDeskBridge.setLocalOption('key')`。 |
| 扫码导入服务器配置后 native key 不同步 | 已修复 | 扫码页保存配置后同步 native key。 |
| 卡片右侧菜单可能穿透触发连接 | 已修复 | 常用/最近会话卡片右侧菜单增加 `HitTestMode.Block`。 |
